An enterprise-grade, asynchronous, and fully decoupled Retrieval-Augmented Generation (RAG) engine engineered to perform accurate question answering and information retrieval over unstructured documentation, college policy guidelines, compliance specs, and high-entropy PDFs (like Code of Conduct, Divyangjan Policy, Anti-Ragging regulations, and Canteen Operations).

This system ensures that answers generated by the Large Language Model (LLM) are strictly backed by actual clauses in your documents, preventing hallucinations and ensuring compliance via **100% verified citation traces**.

---

## 🏗️ System Architecture

The architecture decouples the high-throughput ingestion processor from the query-time neural inference loops to maintain platform isolation and deterministic scaling:

```text
                             [Unstructured Documents Ingestion]
                                             │
                                             ▼
                                    DocumentParserRouter
                       (Unicode Cleansing & Regex Space Normalization)
                                             │
                                             ▼
                                      ChunkingEngine
                       (Adaptive 1500-Character Sliding Windows)
                                             │
                       ┌─────────────────────┴─────────────────────┐
                       ▼                                           ▼
            Dense Embedding Store                       Sparse Inverted Index
        (ChromaDB Local HNSW Graph)                 (Rank-BM25 Pickled Vocabulary)
                       │                                           │
                       └─────────────────────┬─────────────────────┘
                                             │ (Query Parsing Runtime)
                                             ▼
                               Parallel Hybrid Search Retriever
                                             │
                                             ▼
                              Reciprocal Rank Fusion (RRF) Engine
                                             │
                                             ▼
                             Cross-Encoder Neural Re-Ranking Node
                                             │
                                             ▼
                              Groq Synthesis Gateway Engine 
                        (Context-Grounded Llama-3.1 Response + Citations)
```

### Core Pipeline Stages:
1. **Document Ingestion & Sanitization**: Parses raw documents (PDF, HTML, MD, TXT). Strips out corrupted unicode characters and normalizes document layouts.
2. **Dual-Index Ingestion**: Chunks documents into sliding character windows and builds two parallel indices:
   - **Dense Index**: High-dimensional semantic vectors using ChromaDB and `all-MiniLM-L6-v2`.
   - **Sparse Index**: Lexical and vocabulary keyword mappings using `rank-bm25`.
3. **Hybrid Search & Fusion**: Executes parallel lookups across both indices and fuses their rankings using the **Reciprocal Rank Fusion (RRF)** algorithm:
   $$\text{RRF Score} = \sum_{m \in M} \frac{1}{k + r_m(d)}$$
4. **Cross-Encoder Re-Ranking**: Filters out noisy context blocks using the `ms-marco-MiniLM-L-6-v2` cross-encoder to compute query-document cross-attention.
5. **Grounded Generation & Citation Trace**: Synthesizes the final response using Llama 3.1 via Groq JSON Mode, with the **Citation Verifier** auditing generated citations to ensure 100% truthfulness.

## 🛡️ Key Engineering Safeguards

- **Unicode & Regex Sanitizer**: Strips out unpaired surrogates (`\ud800` to `\udfff`) and hidden null bytes (`\x00`) that cause Rust-native Pydantic backend models to crash. It also fixes text extraction bugs in PDFs (like replacing corrupted bullets `` with standard list dashes `-` and fixing lowercase glued words such as `laiddownby` into `laid down by`).
- **Semantic Chunk Deduplication**: Automatically calculates the cosine similarity between chunks using embeddings. If a new chunk is more than 95% similar to an already indexed chunk, it is skipped. This prevents index bloat and saves LLM token costs.
- **Eager Model Warmup**: On system startup, the Streamlit frontend pre-warms the neural Sentence-Transformer and Cross-Encoder weights. This prevents Streamlit from freezing or dropping the WebSocket connection on the first user query.

## 🚀 Quick Start & Installation

### Prerequisites
* Python 3.11 or Python 3.12 (Python 3.14+ is currently unsupported due to missing wheels for core numerical packages).
* A free Groq API Key (get it from the [Groq Console](https://console.groq.com/)).

### 1. Installation
Clone the repository and set up a clean Python virtual environment:

```bash
# Clone the repository
git clone https://github.com/divyyadav007/RAG-Pipeline-with-Hybrid-Search-Over-Internal-Docs.git
cd RAG-Pipeline-with-Hybrid-Search-Over-Internal-Docs

# Create and activate virtual environment
python -m venv .venv
source .venv/bin/activate  # On Windows: .venv\Scripts\activate

# Install dependencies
pip install --upgrade pip
pip install -r requirements.txt
```

### 2. Configure Environment Variables
Copy `.env.example` to `.env` in the root folder of the project:

```bash
cp .env.example .env
```
Inside your `.env` file, configure your API key:
```env
GROQ_API_KEY="gsk_your_groq_api_key_here"
PYTHONPATH="."
```

### 3. Run the Backend API (FastAPI)
Launch the FastAPI microservice that handles ingestion, search, and generation:

```bash
uvicorn src.main:app --reload --port 8000
```
*You can view the interactive API documentation (Swagger UI) at `http://127.0.0.1:8000/docs`.*

### 4. Run the Dashboard UI (Streamlit)
In a new terminal window (with the virtual environment active), start the user interface:

```bash
streamlit run run.py
```
*The dashboard will automatically open in your browser at `http://localhost:8501`.*

To run the entire application inside an isolated Docker container:

```bash

# Run the container
docker run -p 8501:8501 -e GROQ_API_KEY="your_actual_groq_key" hybrid-rag-engine:v1
```

## 📁 Repository Directory Structure

| File / Folder | Purpose |
| :--- | :--- |
| **`src/config.py`** | Central configuration file for model selections, parameters (`CHUNK_SIZE`, `RERANK_TOP_N`), and paths. |
| **`src/main.py`** | FastAPI endpoint definitions for ingestion (`/v1/ingest`) and question answering (`/v1/ask`). |
| **`src/ingestion/`** | Contains document parsers, sliding-window chunkers, Pydantic schemas, and the vector deduplicator. |
| **`src/indexing/`** | Implements the ChromaDB dense index, the pickle-serialized BM25 sparse index, and the hybrid fusion. |
| **`src/reranking/`** | Re-ranks context chunks using the Cross-Encoder transformer model. |
| **`src/generation/`** | Interfaces with Groq models to generate grounded answers and verify citations. |
| **`src/ui/`** | Streamlit user interface code. |
| **`src/evaluation/`** | Holds the automated metrics runner and the 50-sample Golden Test Suite. |
| **`run.py`** | A wrapper script to launch the Streamlit frontend. |
| **`Dockerfile`** | Orchestrates containerization. |
